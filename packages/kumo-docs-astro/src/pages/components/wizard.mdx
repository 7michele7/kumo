## Anatomy

The Wizard is a compound component. Compose only the pieces you need:

```
Wizard.Fullscreen — portal overlay
  └── Wizard.Grid (optional — decorative wireframe)
        └── Wizard — step state, context, and navigation
              └── Wizard.Steps — filters and indexes active steps
                    └── Wizard.Step — individual step with card-stack animation
                          └── Wizard.Page — card surface with title, description, body, and footer
```

</ComponentSection>

## Usage

```tsx
import { Button, Wizard, useWizard, useWizardGrid } from "@cloudflare/kumo";

function CreateWorker({ open, onClose }) {
  const { gridProps, onActiveStepElementChange } = useWizardGrid();

  return (
    <Wizard.Fullscreen open={open} onClose={onClose}>
      <Wizard.Grid title="Create a Worker" {...gridProps}>
        <Wizard onActiveStepElementChange={onActiveStepElementChange}>
          <Wizard.Steps>
            <Wizard.Step stepKey="method" label="Select a method">
              <Wizard.Page
                title="Ship something new"
                description="Choose how you want to start."
                footer={<Footer />}
              >
                {/* ... */}
              </Wizard.Page>
            </Wizard.Step>
            {/* ...more steps */}
          </Wizard.Steps>
        </Wizard>
      </Wizard.Grid>
    </Wizard.Fullscreen>
  );
}

function Footer() {
  const { back, next, isFirstStep } = useWizard();

  return (
    <>
      <Button variant="ghost" onClick={back} disabled={isFirstStep}>
        Back
      </Button>
      <Button onClick={next}>Continue</Button>
    </>
  );
}
```

### `useWizardGrid()`

<p>
  Bridges `Wizard` and `Wizard.Grid` by tracking the active card's height and
  managing transition state. Returns `gridProps` to spread onto `Wizard.Grid`.
</p>

```tsx
import { useWizardGrid } from "@cloudflare/kumo";

const { gridProps, onActiveStepElementChange } = useWizardGrid();

<Wizard.Grid {...gridProps}>
  <Wizard step={step} onStepChange={setStep}>
    <Wizard.Steps>...</Wizard.Steps>
  </Wizard>
</Wizard.Grid>;
```

## API Reference

### Wizard

<p>
  Root context provider and animation engine. Must be rendered inside a
  `Wizard.Fullscreen`.
</p>
<PropsTable component="WizardRoot" />

### Wizard.Fullscreen

<p>
  Fullscreen modal container with a floating close button, Escape-to-close,
  scroll lock, and `role="dialog"`. Content fills the entire viewport.
</p>
<PropsTable component="WizardRoot.Fullscreen" />

### Wizard.Grid

<p>
  Decorative animated wireframe with dashed borders, corner crosses, and an
  optional left-side title. The bottom border and crosses animate with the
  active card's measured height.
</p>
<PropsTable component="WizardRoot.Grid" />

### Wizard.Steps

<p>
  Container for wizard steps. Injects the `index` prop into each child
  `Wizard.Step`.
</p>
<PropsTable component="WizardRoot.Steps" />

### Wizard.Step

<p>
  Individual step definition. Registers metadata with the Wizard context and
  wraps children in a card-stack animation.
</p>
<PropsTable component="WizardRoot.Step" />

### Wizard.Page

<p>
  Card layout inside a wizard step. Wraps content in a `LayerCard` with optional
  title, description, scrollable body, and footer.
</p>
<PropsTable component="WizardRoot.Page" />
